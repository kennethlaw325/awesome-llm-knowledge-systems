## 용어집

> 이 문서는 [English Glossary](../glossary.md)의 한국어 번역입니다. 모든 기술 용어를 일상적인 언어로 설명하므로 박사 학위가 필요하지 않습니다.

### A

**A2A (Agent-to-Agent Protocol, 에이전트 간 프로토콜)**
AI 에이전트들이 서로 소통하고 작업을 조율하기 위한 표준적인 방법. 서로 다른 AI 어시스턴트가 작업을 인계할 수 있게 해 주는 공통 언어와 같습니다.

**Agent Memory (에이전트 메모리)**
AI 에이전트가 대화나 작업을 가로질러 정보를 기억하는 능력. 에이전트가 세션 사이에 보관하는 노트와 같다고 생각하세요. 매번 처음부터 시작할 필요가 없게 만듭니다.

**Agentic RAG (에이전트형 RAG)**
RAG의 한 종류로, AI가 무엇을 찾을지, 언제 찾을지, 결과가 충분히 좋은지를 능동적으로 결정합니다. 매번 같은 고정된 검색 단계를 따르지 않습니다.

**ARC-AGI-3**
François Chollet의 2026년 에이전트 지능을 위한 인터랙티브 벤치마크. 에이전트는 지시 없이 게임 같은 환경에 던져져 스스로 탐험하고, 목표를 추론하고, 세계 모델을 구축해야 합니다. 정적인 퍼즐 그리드로 구성된 이전 ARC 벤치마크와 달리, ARC-AGI-3는 **탐험 효율성**, **목표 추론**, **세계 모델 형성**을 별도의 능력 축으로 평가합니다.

### C

**CATTS (Consensus-Aware Test-Time Scaling, 합의 인식 테스트 시 스케일링)**
멀티스텝 에이전트를 위한 테스트 시 스케일링 방법. 각 단계마다 작은 롤아웃 위원회를 샘플링하고, 위원들 사이의 불일치를 불확실성 신호로 사용하여 컴퓨팅을 할당합니다. 의견이 크게 갈리는 단계는 더 많은 사고 예산을 받고, 합의가 잘 되는 단계는 더 적게 받습니다. 공개된 결과는 균일 스케일링 대비 약 +9.1% 정확도, 2.3배 적은 토큰을 보고합니다.

**Claude Code**
Anthropic의 명령줄 도구. Claude가 터미널에서 직접 작업하면서 파일을 읽고, 명령을 실행하고, 코드를 편집합니다. AI 페어 프로그래밍 동료처럼 동작합니다.

**Codex (OpenAI)**
OpenAI의 도구. AI 에이전트가 샌드박스화된 클라우드 환경에서 코딩 작업을 실행하고, 자율적으로 저장소를 읽고, 코드를 작성하고, 테스트를 실행합니다.

**Context Engineering (컨텍스트 엔지니어링)**
AI가 응답하기 전에 받는 정보를 신중하게 설계하는 실천. Prompt engineering이 질문을 작성하는 것이라면, context engineering은 AI의 책상 위에 어떤 참고 자료를 올려놓을지 선택하는 것입니다.

**Context Window (컨텍스트 윈도우)**
AI 모델이 한 번에 "볼 수 있는" 텍스트의 총량 — 당신의 입력과 모델의 출력 모두를 포함합니다. 화이트보드의 크기와 같다고 생각하세요. 모델이 읽고 쓰는 모든 것이 그 안에 들어가야 합니다.

**CoT Monitoring (사고 연쇄 모니터링)**
모델이 계획을 실행하기 전에 명시적인 추론 토큰을 읽어 잘못된 행동이나 정렬 이탈을 탐지하는 실천. OpenAI는 2026년 4월 자사의 추론 모델 중 하나가 코딩 평가에서 부정행위를 하는 것을 CoT 모니터링으로 적발했습니다. 이는 해석 가능성이 사후 포렌식이 아닌 **런타임 검사**로 작동한 첫 공개 사례입니다.

### E

**Embeddings (임베딩)**
의미가 비슷한 텍스트가 수학적으로도 가까이 위치하도록 텍스트를 숫자 리스트로 변환하는 방법. 컴퓨터가 두 책이 비슷한 주제를 다룬다는 걸 알아채는 것과 같은 방식으로 텍스트의 관련성을 측정할 수 있게 합니다.

**Emotion Vectors (감정 벡터)**
Claude의 내부 활성화 안에 있는 해석 가능한 특징 방향. 이를 강화하면 모델은 감정적 행동에 안정적으로 편향됩니다 — Anthropic의 2026년 4월 공개에서 가장 자주 인용된 예시는 협박성 출력입니다. 이러한 방향이 식별 가능하기 때문에 harness 엔지니어들은 토큰 레벨이 아닌 **특징 레벨**의 필터링 표면을 얻게 됩니다.

### F

**Few-shot Learning (퓨샷 학습)**
모델 전체를 다시 훈련하는 대신 프롬프트 안에 몇 가지 예시를 보여 주어 AI에게 작업 방법을 가르치는 것. 완성된 양식 세 장을 보여 주어 네 번째 양식을 채우는 방법을 알려 주는 것과 같습니다.

**Fine-tuning (파인튜닝)**
사전 훈련된 AI 모델을 자신의 특정 데이터로 추가 훈련하여 특정 작업을 더 잘 수행하게 만드는 것. 제너럴리스트를 채용한 후 전문 OJT를 제공하는 것과 같습니다.

### G

**GraphRAG**
RAG의 한 종류로, 검색된 정보를 연결된 엔티티와 관계의 그래프로 정리하여 여러 출처에서 사실을 결합해야 하는 질문에 더 잘 답할 수 있게 합니다.

### H

**Harness Engineering (하네스 엔지니어링)**
AI 모델을 둘러싼 시스템 — 도구, 메모리, 규칙, 워크플로우 — 전체를 설계하여 모델이 실제로 어떻게 동작할지 형성하는 일. 모델이 엔진이라면, 하네스는 차 전체입니다.

**Harness Synthesis (하네스 합성)**
바깥쪽 루프의 옵티마이저(검색 기반, 관찰성 기반 등)가 대상 작업의 런타임 신호를 바탕으로 하네스 — 그 도구, 프롬프트, 역할 분해, 통신 토폴로지, 협조 프로토콜 — 를 자동으로 수정하는 기술 군. 2026년 4월의 두 참조 구현으로 **AHE**(arXiv 2604.25850)와 **AgentFlow**(arXiv 2604.20801)를 참조하세요. *meta-harness*(2025년 / 2026년 초의 프레이밍, 하네스를 지속적으로 진화하는 산물이 아닌 일회성 최적화 대상으로 다룬 것)와는 구별됩니다.

### I

**Inference (추론)**
AI 모델이 당신의 입력에 대한 응답을 생성하는 과정. 메시지를 보내고 응답을 받을 때마다 모델은 추론을 수행하고 있습니다.

**Iteration Head (이터레이션 헤드)**
사고 연쇄 추론 중에 출현하는 어텐션 헤드로, 이전 추론 단계의 출력에 일관되게 주의를 기울입니다. Anthropic 해석 가능성 팀이 2026년 4월에 식별했습니다. 그 존재는 명시적인 CoT 프롬프팅이 부분적으로는 특정 내부 회로를 유도함으로써 작동하는 메커니즘이며, 단순히 사람이 읽을 수 있는 중간 텍스트를 생성하는 것 이상임을 시사합니다.

### K

**Knowledge Graph (지식 그래프)**
사실의 구조화된 지도로, 엔티티(사람, 장소, 개념)들이 라벨이 붙은 관계로 연결되어 있습니다. 모든 것이 어떻게 서로 연결되어 있는지 보여 주는 라벨링된 끈으로 연결된 색인 카드 웹과 같습니다.

**KV-Cache**
AI가 이전에 계산한 키-값 쌍을 다시 사용해 처음부터 다시 계산할 필요가 없게 해 주는 메모리 단축. 대화 이력이 동일하면 응답이 더 빠르고 저렴해집니다.

### L

**LLM (Large Language Model, 대규모 언어 모델)**
방대한 양의 텍스트로 훈련되어 인간 언어를 이해하고 생성할 수 있는 AI 시스템. ChatGPT, Claude, Gemini는 모두 LLM입니다.

**Long Context (롱 컨텍스트)**
새로운 AI 모델이 한 번에 매우 많은 양의 텍스트를 처리하는 능력 — 책 한 권이나 코드베이스 전체가 한 번의 대화에 들어갈 수도 있습니다.

### M

**MCP (Model Context Protocol, 모델 컨텍스트 프로토콜)**
AI 어시스턴트가 보편적인 플러그 앤 플레이 인터페이스를 통해 외부 도구와 데이터 소스에 연결할 수 있게 해 주는 오픈 표준. AI 애플리케이션을 위한 USB와 같습니다.

**Managed Agents (매니지드 에이전트)**
에이전트 하네스의 기반 — 샌드박스, 세션 상태, 범위가 지정된 도구 실행, 트레이싱 — 을 개발자가 아닌 모델 벤더가 운영하는 호스팅 런타임 모델. Anthropic은 2026년 4월 8일 첫 상용 사례를 퍼블릭 베타로 출시했습니다([platform.claude.com/docs/en/managed-agents/overview](https://platform.claude.com/docs/en/managed-agents/overview)). SiliconANGLE의 출시 보도에 따르면, 가격은 표준 API 토큰 요율에 더해 기반에 대한 에이전트 런타임 시간당 요금입니다(시간당 수치는 Anthropic의 1차 문서에 기재되어 있지 않습니다). Claude Code Routines와 같은 클라우드 네이티브 트리거 표면과는 구별됩니다. 후자는 Managed Agents 식의 기반 위에 올라가지만 다른 질문에 답합니다 — "루프가 무엇으로 시작되는가".

**Mechanistic Interpretability (메커니즘적 해석 가능성)**
모델의 가중치 내부에 있는 인간이 이해할 수 있는 회로 — 특정 행동을 구현하는 특징, 헤드, 경로 — 를 식별하는 것을 목표로 하는 연구 프로그램. MIT Technology Review의 2026년 10대 돌파 기술 중 하나로 선정되었으며, 2026년 4월의 감정 벡터, 이터레이션 헤드, CoT 모니터링 같은 결과를 뒷받침합니다.

**mHC (Manifold-Constrained Hyper-Connections, 매니폴드 제약 하이퍼 연결)**
DeepSeek의 2026년 4월 아키텍처 제안으로, 잔차 연결을 학습된 저차원 매니폴드를 따라 여러 내부 정보 스트림을 라우팅하도록 확장합니다. Transformer++ 스타일 모델의 단일 잔차 스트림을 여러 협조하는 스트림으로 일반화한 것입니다. 발표 시점에서 결과는 **독립적인 재현을 기다리는 중**으로 표시되어 있습니다.

**MIRAS**
Google Research의 메모리 증강 훈련 프레임워크로, 메모리가 추론 시에 업데이트되는 훈련 가능한 신경 모듈인 Titans 같은 아키텍처를 위한 레시피, 안정성 보장, 학습 동역학을 제공합니다. MIRAS는 메모리 모듈이 발산하지 않으면서 "추론하면서 학습"을 실용적으로 만드는 프레임워크입니다.

**MoE (Mixture of Experts, 전문가 혼합)**
주어진 입력에 대해 모델의 "두뇌"의 일부만 활성화되는 모델 아키텍처. 모든 부분이 매번 실행되지 않기 때문에 매우 큰 모델을 만들면서도 빠른 속도를 유지할 수 있습니다.

### O

**Obsidian**
노트를 자신의 컴퓨터에 일반 텍스트 파일로 저장하고 서로 연결하여 개인 지식 베이스를 만들 수 있게 해 주는 노트 앱.

### P

**Progressive Disclosure (점진적 공개)**
필수 정보만 먼저 보여 주고 필요할 때 더 자세한 내용을 드러내는 디자인 원칙. FAQ 페이지처럼 질문이 보이고, 실제로 필요한 답만 클릭으로 펼쳐 봅니다.

**Prompt Engineering (프롬프트 엔지니어링)**
가능한 최선의 응답을 얻을 수 있도록 AI 모델에 지시문을 작성하는 기술. 표현의 작은 변화가 매우 다른 결과를 낳을 수 있습니다.

### R

**RAG (Retrieval-Augmented Generation, 검색 증강 생성)**
AI가 답하기 전에 외부 출처에서 관련 정보를 찾아보고, 훈련 시 외운 내용에만 의존하지 않고 실제 데이터에 근거한 응답을 하는 기술.

**Routines (Claude Code)**
Anthropic의 2026년 4월 Claude Code용 클라우드 네이티브 하네스 프리미티브. 에이전트 워크플로우는 사용자의 머신이 아닌 Anthropic 클라우드에서 실행되며, 스케줄, API 호출 또는 GitHub 이벤트로 트리거할 수 있습니다. Routines는 자체 호스팅 cron+daemon 패턴을 매니지드 기반으로 일반화하며 티어별 쿼터(Pro 5/일, Max 15/일, Team/Enterprise 25/일)를 갖추고 사용자의 노트북이 오프라인일 때도 계속 작동합니다.

### S

**Self-RAG**
RAG의 한 종류로, AI가 자신이 가져온 정보와 생성한 답변의 품질을 평가하여 추가 검색이 필요한지 또는 응답을 수정해야 하는지를 결정한 후 최종 결과를 제공합니다.

**Session-hour pricing (세션 시간당 과금)** (*agent-runtime-hour pricing*으로도 보도)
오케스트레이터 자리 — 에이전트 루프가 실행되는 기반 — 를 추론과 별도로 계량하는 과금 모델. Anthropic Managed Agents가 2026년 4월 8일 처음 상용 도입했습니다. SiliconANGLE 출시 보도에 따르면 표준 토큰 요율에 더해 에이전트 런타임 시간당 8센트입니다. 시간당 수치는 Anthropic의 1차 문서에 기재되어 있지 않으며 2차 테크 프레스에서 인용된 것입니다. 주목할 점은 이것이 "루프가 어디에서 실행되는가"의 비용에 숫자를 붙인 **첫 번째 벤더 프리미티브**라는 점이며, "루프가 무엇을 생각하는가"의 비용과 구별됩니다. 정확한 표현은 1차 문서(*sessions* 사용)와 테크 프레스 보도(*agent runtime hour* 사용) 사이에서 다르지만 둘 다 같은 계량 표면을 가리킵니다.

**Skill (AI Agent Skill)**
AI 에이전트가 호출할 수 있는 재사용 가능하고 패키지화된 능력 — "이 PR을 리뷰하라"나 "데일리 리뷰를 실행하라" 같은 특정 작업을 위한 레시피와 같습니다.

**Skill Graph (스킬 그래프)**
AI 에이전트가 사용할 수 있는 모든 스킬의 지도. 스킬들이 서로 어떻게 관련되어 있는지, 각 스킬이 언제 트리거되어야 하는지를 포함합니다.

**System Prompt (시스템 프롬프트)**
당신의 대화가 시작되기 전에 AI 모델에 주어지는 숨겨진 지시문으로, 모델의 역할, 규칙, 행동을 설정합니다. 직원이 첫 출근 전에 읽는 직무 기술서와 같습니다.

### T

**Task Budget (태스크 버짓)**
Anthropic이 Claude Opus 4.7에서 도입한 하네스 프리미티브(2026년 4월, beta 헤더 `anthropic-beta: task-budgets-2026-03-13`). 호출자는 **에이전트 루프 전체**(사고, 도구 호출, 도구 결과, 최종 출력)에 대한 권고적 토큰 예산을 선언하고, 모델은 작업하는 동안 카운트다운을 받아 한 단계가 검색, 추론, 종합에 얼마나 더 들일 가치가 있는지 결정합니다. 모델에 보이지 않는 하드 캡인 `max_tokens`와는 다릅니다. 예산은 강제가 아닌 권고적이며, 빠듯한 예산에서 거부 응답으로 퇴화하는 것을 막기 위해 최소 20K 토큰이 보장됩니다. Task budgets는 "각 단계에 대해 얼마나 깊이 생각해야 하는가"를 수동 조정 매개변수가 아닌 **매니지드 컨트랙트**로 표면화한 첫 번째 벤더 프리미티브입니다.

**Titans**
Google Research의 2026년 4월 아키텍처 제품군. 메모리는 외부 벡터 저장소나 고정된 어텐션 윈도우가 아닌, 추론 시 경사 하강법으로 자신을 업데이트하는 훈련 가능한 신경 모듈입니다. 비교 가능한 매개변수 수에서 Titans는 장거리 회상과 멀티홉 추론 벤치마크에서 Mamba-2, Gated DeltaNet, Transformer++을 능가하는 것으로 보고되어, "컨텍스트"와 "파인튜닝" 사이의 경계를 흐립니다.

**Token (토큰)**
AI 모델이 읽고 쓰는 기본 단위 — 영어로는 대략 단어의 4분의 3에 해당합니다. 모델은 토큰 단위로 사고하며, 가격과 컨텍스트 한도는 토큰으로 측정됩니다.

**Tool Use / Function Calling (도구 사용 / 함수 호출)**
AI 모델이 텍스트 생성뿐 아니라 외부 행동 — 웹 검색, 코드 실행, API 호출 — 을 트리거할 수 있는 능력.

### V

**Vector Database (벡터 데이터베이스)**
임베딩을 저장하고 검색하기 위해 전용으로 설계된 데이터베이스. 수백만 항목 중에서 가장 유사한 항목을 빠르게 찾을 수 있게 합니다. 대부분의 RAG 시스템의 엔진입니다.

### W

**Wikilink**
Obsidian 같은 도구에서 사용되는 이중 대괄호 링크(예: `[[노트 제목]]`). 한 노트를 다른 노트에 연결하여 연결된 지식의 웹을 만듭니다.
